# HTA – Consulta ao Histórico de Processamentos
## Tabela de Problemas e Recomendações

| Objetivos / Operações | Problemas e Recomendações |
|---|---|
| **0. Consultar histórico de processamentos (1 → 2 → 3)** | **Input:** registros de processamentos anteriores armazenados no sistema <br> **Feedback:** lista de resultados com data, área analisada e opções de acesso <br> **Plano:** acessar dashboard inicial → consultar histórico → utilizar resultado ou iniciar novo processo <br> **Recomendação:** apresentar histórico organizado e fácil de localizar |
| **1. Acessar o Dashboard inicial (1.1 → 1.2)** | **Plano:** abrir sistema → visualizar tela Home <br> **Problema:** usuário pode ter dificuldade em localizar o histórico na interface <br> **Recomendação:** destacar seção de histórico na página inicial |
| **1.1 Abrir sistema** | **Problema:** carregamento inicial pode demorar ou gerar incerteza <br> **Recomendação:** exibir indicador de carregamento |
| **1.2 Visualizar tela Home** | **Problema:** excesso de elementos na interface pode dificultar a navegação <br> **Recomendação:** organizar layout com foco nas informações principais |
| **2. Consultar histórico (2.1 / 2.2 / 2.3)** | **Plano:** visualizar lista de processamentos → verificar informações → identificar processamento desejado  <br> **Problema:** grande quantidade de registros pode dificultar a busca <br> **Recomendação:** oferecer filtros e ordenação por data |
| **2.1 Visualizar lista de processamentos recentes** | **Problema:** lista longa pode dificultar leitura <br> **Recomendação:** usar paginação ou limitar número de itens exibidos |
| **2.2 Visualizar data e área analisada** | **Problema:** informações podem não estar claras para o usuário <br> **Recomendação:** usar rótulos claros e formatação padronizada |
| **2.3 Identificar processamento desejado** | **Problema:** usuário pode selecionar o item errado <br> **Recomendação:** exibir miniatura ou descrição do processamento |
| **3. Utilizar histórico (3.1 → 3.2 → 3.3)** | **Plano:** reabrir resultado anterior → visualizar mapas e estatísticas → iniciar novo processo se necessário <br> **Problema:** usuário pode não saber quais ações pode realizar após abrir o resultado <br> **Recomendação:** apresentar opções claras de ação |
| **3.1 Reabrir resultado anterior** | **Problema:** carregamento do resultado pode demorar <br> **Recomendação:** exibir indicador de progresso |
| **3.2 Visualizar mapas e estatísticas** | **Problema:** excesso de dados pode dificultar interpretação <br> **Recomendação:** destacar informações principais |
| **3.3 Iniciar novo processo se necessário** | **Problema:** opção de iniciar novo processamento pode não estar clara <br> **Recomendação:** incluir botão visível de “Novo processamento” |
